# Координация AI-агентов в этом репо

В проекте работают **несколько AI-агентов** одновременно. Чтобы они не перезаписывали работу друг друга — каждому жёстко ограничена своя зона. Любой агент перед коммитом ОБЯЗАН прочитать этот файл и не трогать чужие зоны.

## Агент 1 — Writer (Cowork Claude / Дарья пишет в чате)

Отвечает за **визуал, статьи и архитектуру сайта**.

**Может менять:**
- `_guides/**` — содержимое всех гайдов
- `_layouts/**` — шаблоны страниц
- `_includes/**` — переиспользуемые HTML-блоки (header, footer, banner)
- `assets/**` — CSS, изображения, иконки
- `index.html`, `ru/**`, `_config.yml` — структура сайта
- `prompts/style-guide-jlm.md` — методичка для написания
- `CLAUDE.md`, `AGENTS.md` — общие документы

**Не трогает:**
- `research/**` — это зона Researcher
- `prompts/ru-topic-researcher.md` — это конфиг Researcher

## Агент 2 — Researcher (поиск тем по форумам и Wordstat)

Отвечает за **поиск новых тем** и сохраняет результаты в виде markdown-таблиц.

**Может менять:**
- `research/**` — новые темы, исследования, цитаты клиентов
- `prompts/ru-topic-researcher.md` — улучшения собственного промпта (если осознанно)

**КАТЕГОРИЧЕСКИ не трогает:**
- `_guides/**` — статьи пишет Writer (даже если кажется что можно «улучшить» — НЕ ТРОГАЙ, отдай Writer-у через research-таблицу)
- `_layouts/**`, `_includes/**` — шаблоны пишет Writer
- `assets/**` — стили и картинки пишет Writer
- `index.html`, `ru/**` — структуру сайта пишет Writer
- `_config.yml` — настройки Jekyll меняет только Writer
- `CLAUDE.md`, `prompts/style-guide-jlm.md` — документы Writer-а

## Что делать, если Researcher хочет повлиять на статьи

Не редактировать! Вместо этого — добавить в `research/<тема>.md` рекомендацию для Writer-а:

```markdown
## Рекомендации Writer-у
- В существующей статье `objection-dorogo` стоит добавить пример с возражением «у нас нет бюджета в этом квартале» — встречается в N постах, цитата: «...»
```

## Перед коммитом — обязательная проверка

```bash
# Researcher проверяет что не залез в чужую зону:
git diff --name-only HEAD | grep -vE '^(research/|prompts/ru-topic-researcher\.md$)' && echo "ОШИБКА: ты тронул чужие файлы. Откати их через git checkout."

# Writer аналогично:
git diff --name-only HEAD | grep -E '^(research/)' && echo "ОШИБКА: research/ — зона Researcher, откати."
```

Если зацепил чужой файл случайно — `git checkout HEAD -- <файл>` и не коммить его.

## Если зоны пересекаются по делу — координация через коммит-сообщения

Researcher: коммит-сообщение начинается с `[research]`
Writer: коммит-сообщение начинается с `[content]` или `[design]`

Перед началом работы — `git pull origin main` и `git log --oneline -5` чтобы увидеть, кто что делал недавно.
