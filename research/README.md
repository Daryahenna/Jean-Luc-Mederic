# research/

Здесь живёт автоматический Reddit-ресерчер и его выгрузки.

## Как это работает

`reddit_research.py` ходит по выбранным subreddit-ам (`r/sales`, `r/Entrepreneur`, `r/smallbusiness`, `r/freelance`, `r/consulting`), ищет посты и комментарии по ~17 запросам про продажи/возражения/AI/закрытие, фильтрует по upvotes (≥5) и комментариям (≥3), сохраняет:

- `reddit_results.json` — полный архив постов (с дедупликацией по URL)
- `reddit_digest.md` — человекочитаемый дайджест топ-30 постов с цитатами

В GitHub Actions запускается **3 раза в сутки** (06:00, 14:00, 22:00 UTC) автоматически — workflow `.github/workflows/reddit-research.yml` коммитит обновления обратно в main, если что-то новое нашлось.

## Один раз настроить

**Ничего не нужно настраивать.** Скрипт использует публичный Reddit JSON API — без OAuth, без секретов, без developer-аккаунта. Всё, что нужно — это сам файл (уже в репо) и workflow (тоже в репо).

Опционально: проверить, что в Settings → Actions → General → Workflow permissions выставлено «Read and write permissions». Без этого workflow не сможет коммитить обновлённый дайджест обратно.

## Почему так

Reddit с 2023 года ужесточил Responsible Builder Policy — для создания нового API-приложения нужна отдельная заявка с одобрением. Чтобы не зависеть от их регистраторов, скрипт работает через **публичный JSON-эндпойнт**, который Reddit отдаёт без auth (тот самый, что использует старый Reddit-фронтенд). Лимит — ~60 запросов в минуту с одного IP, мы держим ~50, с запасом.

## Как запустить локально (для теста)

```bash
pip install requests
python research/reddit_research.py
```

Результаты появятся в `research/reddit_results.json` и `research/reddit_digest.md`.

## Как использовать собранные данные

Раз в неделю открывай `reddit_digest.md`, смотри топ постов:
- Если боль/тема **повторяется в 3+ постах** — это P1 для статьи
- Если есть **сильная цитата** (живая, эмоциональная, с подробностями) — это материал для intro новой статьи или для blockquote
- Если **новая тема** (не покрыта в `topics-ru-vozrazheniya.md` / `topics-ru-batch2.md`) — добавляй в новый `topics-ru-batchN.md`

В перспективе можно автоматизировать и эту часть: AI читает digest, обновляет topics-batch автоматически, флагает новые темы. Но это уже после 9–10 мая.

## Файлы в этой папке

- `reddit_research.py` — скрипт-сборщик (публичный JSON API, без секретов)
- `reddit_results.json` — архив всех найденных постов (растёт со временем)
- `reddit_digest.md` — топ-30 свежих постов в читаемом виде, обновляется автоматически
- `topics-ru-vozrazheniya.md` — первая партия тем (10 шт, кластер «Возражения»)
- `topics-ru-batch2.md` — вторая партия (10 шт, разные кластеры)
- `_tmp_fetch.py` — отдельный helper для Bing/Bukvarix (не относится к Reddit, можно не трогать)
